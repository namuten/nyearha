# 프로젝트 규칙 (Rules)

이 문서에 적힌 규칙은 **모든 작업의 절대 제약**이다. 위반 불가.

---

## 1. 시험 범위 준수

- `범위.md`에 정의된 시험 범위는 **변경 불가능한 고정값**이다.
- 모든 분석/요약/문제 생성은 이 범위를 정확히 커버해야 한다.
- 범위 밖의 내용은 다루지 않는다.

---

## 2. 기출 문제 생성 3단계 프로세스

각 단계의 산출물은 **반드시 별도의 MD 파일**로 저장한다.

### STEP 1 — 교과서 분석 (역할: 교과서 분석 전문가)

**입력:** 교과서/학습지 PDF (OCR 기반 분석)

**작업:**
1. 모든 텍스트를 빠짐없이 추출
2. 표는 반드시 구조를 유지해서 정리
3. 그림/도표는 내용 추론해서 설명 추가
4. 핵심 개념은 별도로 표시

**출력 형식(JSON, MD 파일 내 코드 블록으로):**
```json
{
  "chapters": [
    {
      "title": "",
      "concepts": [],
      "details": "",
      "tables": [],
      "figures": [
        {
          "description": "",
          "related_concept": ""
        }
      ]
    }
  ]
}
```

**산출 파일:** `step1_<과목>_분석.md`

---

### STEP 2 — 출제 포인트 정리 (역할: 출제위원)

**입력:** STEP1 산출물

**작업 (조건):**
- 암기형 / 이해형 / 계산형 구분
- 자주 틀리는 포인트 강조
- 표/그림 기반 문제 가능 포인트 따로 표시

**출력 항목:**
1. 핵심 개념 요약
2. 출제 포인트 리스트
3. 함정 포인트

**산출 파일:** `step2_<과목>_출제포인트.md`

---

### STEP 3 — 실제 문제 생성 (역할: 시험 출제자)

**입력:** STEP2 산출물

**작업 (조건):**
- 객관식 5지선다
- 난이도: 쉬움 / 보통 / 어려움 섞기
- 표/그래프 해석 문제 반드시 포함
- 단순 암기 문제 금지

**출력 항목 (문제마다):**
- 문제
- 보기 (20개)
- 정답
- 해설 (왜 틀리는지 포함)

**산출 파일:** `step3_<과목>_문제.md`

---

## 3. 파일 명명 규칙

- 모든 산출물은 한글 파일명 사용 가능
- `step<번호>_<과목>_<용도>.md` 형식 권장
- 과목명은 `범위.md`의 교과목명과 일치 (예: 공통국어1, 통합과학1, 통합사회1, 한국사1, 공통수학1, 공통영어1)
